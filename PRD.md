這是一份針對 Python Flask 製作「ERP 子模組 DEMO 網站」的 **PRD (Product Requirement Document)**，重點是單機 Flask 網站，不需 API、資料庫，以靜態資料及 session 模擬流程。  

***

# ERP DEMO 網站 PRD (Product Requirement Document)

## 1. 產品概述
此專案是一個簡化版的 ERP 系統 DEMO 網站，使用 **Python Flask** 實作全端單體架構，不依賴資料庫或外部 API。主要展示 ERP 系統中不同角色、模組及流程的操作，以方便教學或展示內部流程邏輯。  
系統資料以 **記憶體（dictionary, list）或 session 模擬**，報表以動態渲染 HTML Table 呈現。

***

## 2. 核心目標
- 展示 ERP 子模組（進銷存、訂單管理）流程。
- 模擬多角色（採購員、倉管員、財務）操作，不同權限進入不同頁面。
- 展現資料流轉（如訂單 → 進貨 → 出貨 → 報表）。
- 提供 DEMO 純網頁化，不依賴 API 或 DB，降低部署與學習門檻。

***

## 3. 系統架構
- **框架**：Python Flask (單體架構，全站 routing)
- **前端**：HTML + Bootstrap（簡單響應式介面）
- **資料儲存**：Flask session / Global Dictionary（模擬暫存）
- **身分與授權**：
  - 管理員（可檢視所有模組）
  - 採購員（可新增訂單、上傳進貨紀錄）
  - 倉管員（可更新庫存、處理出貨）
  - 財務員（可檢視損益報表、付款狀態）

***

## 4. 功能模組  

### 4.1 使用者登入與權限
- 登入頁：輸入帳號（無需註冊，Demo 固定帳號）
- 判斷角色，導向不同功能首頁
- Session 模擬登入狀態

### 4.2 訂單管理（銷售流程）
- 建立訂單（輸入產品、數量、單價）
- 訂單清單（顯示狀態：已建立、已出貨、已付款）
- 訂單查詢（依產品或客戶名字模擬）

### 4.3 採購管理
- 新增採購單（供應商、商品、數量、成本）
- 採購清單與狀態更新（待交貨、已入庫）

### 4.4 庫存管理
- 商品清單（名稱、編號、庫存數量）
- 入庫（來自採購單）
- 出庫（來自銷售訂單）
- 即時庫存報表

### 4.5 財務報表
- 銷售毛利（以訂單價格 - 採購成本）
- 銷貨收入統計
- 庫存價值總額

### 4.6 系統管理
- 用戶角色設定（預設帳號）
- 系統初始化（清空暫存資料）

***

## 5. 頁面設計概要
- `/login` 登入頁
- `/dashboard` 主控台（依權限顯示模組入口）
- `/orders` 訂單列表 / 新增訂單
- `/purchase` 採購單 / 入庫操作
- `/inventory` 庫存查詢 / 出庫功能
- `/finance` 財務報表
- `/admin` 系統管理

***

## 6. Demo 流程範例
1. 採購員 → 登入 → 建立採購單 (100件商品A) → 狀態：待交貨  
2. 倉管員 → 入庫操作 → 商品A +100 → 成功入庫  
3. 採購單狀態變更 → 已入庫  
4. 銷售員 → 建立訂單 (客戶買50件商品A) → 系統檢查庫存足夠 → 出貨 → 扣庫存  
5. 財務 → 查詢損益報表，顯示收入、成本、庫存價值

***

## 7. 成功指標
- 使用者可體驗一個「完整 ERP 子流程」：採購 → 入庫 → 銷售 → 報表
- 成功展示多角色登入與操作差異
- 整體系統可離線運行（無需外部 API 或 DB）
